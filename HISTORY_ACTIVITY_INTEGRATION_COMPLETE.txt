ACTIVITY LOG DATA STRUCTURE
───────────────────────────────────────────────────────────────────────────────

Backend Schema (ActivityLog table):
- id: int (primary key)
- user_id: int (foreign key)
- action: str (Created, Updated, Deleted, Generated, etc.)
- entity_type: str (Invoice, Customer, Vendor, Purchase Order, etc.)
- entity_id: str (optional - ID or number of affected entity)
- title: str (Human-readable event title)
- description: str (Full description sentence)
- created_at: datetime (UTC timestamp)

Response Format (Backend → Frontend):
{
  "success": true,
  "data": [
    {
      "id": 1,
      "action": "Created",
      "entityType": "Invoice",
      "entityId": "INV-0042",
      "title": "Invoice Created",
      "description": "Invoice INV-0042 was created successfully.",
      "createdAt": "2026-07-08T14:00:00.000Z"
    }
  ]
}

Frontend Computed Fields:
- subtitle: Uses description or "action entityType"
- time: Formatted relative time (e.g., "2 hours ago")
- icon: Selected based on entity type + action
- color: Selected based on action type
