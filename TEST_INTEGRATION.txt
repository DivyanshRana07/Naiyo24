========================================
TROUBLESHOOTING
========================================

Problem: Backend not responding
Solution: 
  docker-compose ps   (check if running)
  docker-compose logs backend   (check errors)
  docker-compose restart backend

Problem: Empty response from backend
Solution:
  curl http://localhost:8000/api/v1/customers
  Check if returns empty array []

Problem: Flutter can't connect
Solution:
  Check API URL in api_routes.dart
  Should be: http://localhost:8000/api/v1
  NOT: http://127.0.0.1:8000/api/v1

Problem: Seed data still appearing
Solution:
  Clear app cache
  Clear browser data
  Check notifiers are updated

Problem: PDF not downloading
Solution:
  Check browser console for errors
  Check backend logs
  Verify invoice exists in database

DOCKER COMMANDS REFERENCE
========================================

Status:     docker-compose ps
Start:      docker-compose up -d
Stop:       docker-compose stop
Logs:       docker-compose logs -f
DB Access:  docker-compose exec db psql -U invoice_user -d invoice_business_tools
Backend:    docker-compose exec backend bash
Restart:    docker-compose restart backend
Rebuild:    docker-compose up -d --build
Clean:      docker-compose down -v
